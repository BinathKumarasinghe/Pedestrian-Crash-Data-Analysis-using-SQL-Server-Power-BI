📊 Pedestrian Crash Data Analysis

This project presents a comprehensive analysis of pedestrian crash data using SQL Server and Power BI, aiming to uncover meaningful patterns and insights to support road safety improvements.

🔍 Project Overview

Understanding pedestrian crash behavior is critical for reducing accidents and enhancing public safety. This project involves data preprocessing, cleaning, and visualization to identify trends related to crash frequency, severity, time distribution, and geographic hotspots.

⚙️ Technologies Used
SQL Server (SSMS) – Data storage, querying, and preprocessing
Power BI – Data visualization and dashboard development
DAX – Calculated measures and data modeling
🧹 Data Processing Workflow
Data Importing
Dataset imported into SQL Server database (Crashed_Data)
Structured into Pedestrian_Crashes table
Data Cleaning
Handled missing values in key fields (date, time, severity)
Removed duplicate records
Corrected datatype inconsistencies
Data Transformation
Converted date and time fields into appropriate formats
Standardized categorical values (e.g., crash severity, crash type)
📈 Dashboard & Visualizations

The Power BI dashboard includes:

Total casualties and crash count (KPI cards)
Crash trends over time (year-wise analysis)
Crash distribution by time of day
Severity analysis (fatal, injury, property damage)
Crash type breakdown
Geographic mapping of crash locations
Interactive filters and slicers for deeper exploration
🔑 Key Insights
Majority of crashes resulted in injuries, with relatively few fatal cases
Peak crash occurrences observed during afternoon and rush hours
“Struck Pedestrian” identified as the most frequent crash type
Certain locations showed higher crash density, indicating high-risk zones
🎯 Conclusion

The analysis highlights critical patterns in pedestrian crashes and demonstrates how data-driven insights can assist policymakers, urban planners, and safety authorities in implementing targeted interventions to reduce accidents and improve road safety.
